# 报告自检清单

输出最终执行报告前，确认以下事项：

- [ ] 没有修改 `design-docs/` 下的任何文件。
- [ ] 没有修改 `test-cases/`；即使为本地实验临时修改，也不得把这些改动纳入最终补丁。
- [ ] 如果需要新增黑盒验收，用例已写入 `test-case-new/`，而不是写入原始 `test-cases/`。
- [ ] `test-case-new/` 的期望来自 `design-docs/` 或 README REST 契约，而不是来自当前实现。
- [ ] 已生成并审核规格清单、设计侧契约和规格追踪矩阵。
- [ ] 设计候选清单中的每一项都保留在审核后清单，且 `informational` 项具有具体理由。
- [ ] 每个冻结 REST 接口映射到发出真实 HTTP 请求且包含有效断言的黑盒测试。
- [ ] 路由探测没有计入精确成功状态码和业务语义覆盖。
- [ ] 每条追踪映射的专属证据和断言实际位于所引用的测试方法体内，而不是文件级注释。
- [ ] 没有使用只枚举端点或只断言列表数量的元测试冒充黑盒覆盖。
- [ ] 状态机、公式、权限、事件和架构规则已使用适合的方法验证，或明确记录未验证风险。
- [ ] 冻结 API 的路径、HTTP Method、字段名、字段类型、成功状态码和 `/api/v1/` 前缀均未改变。
- [ ] 每一项行为变更都能追溯到设计文档或 README API 契约。
- [ ] 错误码和 HTTP 状态码与 README 第 7 节一致。
- [ ] 已运行模块测试、原始黑盒测试和 `test-case-new` 黑盒测试；若因环境限制无法运行，已在报告中说明。
- [ ] 追踪校验脚本通过，报告中的覆盖率可由规格 ID 和测试文件复核。
- [ ] 修复黑盒失败时已同步新增或更新根因 UT。
- [ ] 开始和结束时的受保护文件摘要一致。
- [ ] 残余风险列出了尚未验证的模块或无法运行的命令。
